# ADR-0001: 日本株 Alphalens 研究のハーネス方針

- Status: Accepted
- Date: 2026-03-20
- Deciders: リポジトリメンテナ
- Supersedes: None

## 背景

リポジトリ内のガイダンスが古い、または実行不能な場合、コーディングエージェントはドリフトを増幅させます。  
この日本株 Alphalens プロジェクトを安定運用するために、最小実行可能ハーネスが必要です。

## 決定

決定論的な品質ループを強制し、実行可能なチェックを契約として扱います。

すべての変更で必須とするループ:

1. `uv run ruff format .`
2. `uv run ruff check .`
3. `uv run pytest -q`

あわせて次を採用します:

- ポインタ型 `AGENTS.md`（短く保ち、長文方針は書かない）
- `docs/templates/` のテンプレートによる Plan/Execute 分離
- 継続的に参照できるアーキテクチャ判断のための ADR
- 確認済みバグごとの回帰テスト追加

## 影響

良い影響:

- エージェントのフィードバックループが速く安定する
- レビュー時のスタイル差分が減る
- セッションをまたいだ回帰リスクが下がる

トレードオフ:

- 変更ごとのローカル検証時間はやや増える
- テストと lint ルールの継続メンテナンスが必要

## 準拠条件

lint と tests が通らない変更は完了とみなしません。  
方針更新は、この ADR を supersede する新しい ADR を追加して行います。
